# Session: 2026-03-24

## Goal
Design per-project static website generator integrated into Directions workflow, complementing existing ProjectOverview aggregator.

## Progress
- Explored existing ProjectOverview at `3-Websites/ProjectOverview/` (SvelteKit, collector script, 6 views)
- Analyzed all Directions file structures available for parsing (PROJECT_STATE, decisions, sessions, cookbook)
- Designed bottom-up architecture: each project generates `docs/site/index.html` + `project.json`
- Identified relationship: ProjectOverview = top-down aggregator, new system = bottom-up per-project sites
- Proposed integration points: `/setup` (initial generation), `/log` (auto-regenerate)

## Decisions
- Per-project sites will be self-contained static HTML (zero dependencies, inline CSS)
- Each site also generates `project.json` as a contract for the master aggregator
- Master aggregator lives in `0-DIRECTIONS/__DIRECTIONS/site/`
- Auto-triggers: generate on `/setup`, update on `/log`
- Dark minimal style consistent with user's app aesthetic

## Architecture (Proposed)
```
Per-project:
  docs/site/index.html     ← self-contained dark minimal page
  docs/site/project.json   ← structured data for aggregator

Master aggregator (0-DIRECTIONS):
  __DIRECTIONS/site/
    index.html              ← discovers all project.json files
    collect.js              ← scans sibling project folders
```
